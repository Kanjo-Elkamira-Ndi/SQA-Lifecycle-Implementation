# 🧪 Software Quality Assurance (SQA) Lab Repository

## 📌 Overview

This repository contains the complete implementation of the **Software Quality Assurance (SQA) Continuous Assessment Labs**.

The project demonstrates practical application of software quality principles including:

* ✅ Unit Testing
* ✅ Integration Testing
* ✅ Version Control & Configuration Management
* ✅ Git Branching Strategy (GitFlow Simulation)
* ✅ Release Management & Tagging
* ✅ Performance Testing

The objective is to simulate **real-world software engineering workflows** while applying formal SQA practices.

---

# 🎯 Purpose of the Repository

This repository was created to:

* Demonstrate structured testing using **Node.js and Jest**
* Apply professional Git version control practices
* Implement a layered branching strategy (`main`, `develop`, feature branches)
* Simulate enterprise-style release management
* Validate functional correctness and system reliability
* Practice configuration management discipline

It reflects a complete **Software Development Lifecycle (SDLC) quality workflow**.

---

# 🧱 Repository Structure

```bash
.
├── Lab01/        # Unit Testing (Business Logic Validation)
├── Lab02/        # Integration Testing
├── Lab03/        # Git Configuration & Version Control Management
├── main          # Stable production-ready branch
├── develop       # Integration branch
```

Each lab builds upon the previous one to progressively introduce deeper quality assurance practices.

---

# 🧪 Lab Breakdown

## 🔹 Lab 01 – Unit Testing

* Implementation of core business logic modules
* Test-driven validation using **Jest**
* Boundary testing
* Edge case handling
* Defensive programming

📍 Focus: Functional correctness at component level.

---

## 🔹 Lab 02 – Integration Testing

* Integration of multiple modules
* Verification of module interaction
* End-to-end behavior validation
* Error propagation testing

📍 Focus: System behavior across components.

---

## 🔹 Lab 03 – Configuration Management (Git)

* Repository initialization and remote setup
* Feature branch creation
* `develop` branch integration
* Merging strategy
* Conflict resolution
* Release tagging (`v1.0.0`)

📍 Focus: Professional source code management practices.

---

# 🔀 Branching Strategy

This repository follows a structured branching approach:

* `main` → Production-ready stable code
* `develop` → Integration branch
* `lab01`, `lab02`, etc. → Feature branches

This simulates a simplified **GitFlow workflow** used in enterprise environments.

---

# 🏷 Versioning

Semantic versioning is used for releases:

```
vMAJOR.MINOR.PATCH
```

Example:

```
v1.0.0 → Initial stable release after Labs 1 & 2 integration
```
